# Frontend Instructions

## Purpose
The frontend is a control panel for humans.
It exists to provide:
- transparency
- trust
- explicit control

## Rules
- React + Next.js + TypeScript only
- Functional components only
- No business logic in components
- No direct API calls in UI components
- Use a service layer for all backend communication

## UX Principles
- Relevance checkbox = explicit permission
- All automated actions must be visible
- No hidden background behavior
- Status must always be understandable

## Error Handling
- Show meaningful error messages
- Never silently fail
